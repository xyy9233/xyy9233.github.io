0. 概述

什么是虚拟机，什么是VM？

虚拟机：自己定义一套指令，在程序中能有一套函数和结构，解释自己定义的指令并执行功能。

VM：一种基于虚拟机的代码保护技术。将基于x86汇编系统中的可执行代码转换为字节码指令系统的代码，来达到不被轻易篡改和逆向的目的。

被攻击危害：vm题目通常在服务器上部署，被攻击后可能会面临系统崩溃、数据泄露、权限提升、服务中断、恶意代码执行。

1. 什么是虚拟机，什么是VM？

虚拟机：自己定义一套指令，在程序中能有一套函数和结构解释自己定义的指令并执行功能。

比如我们之前做过的NEMU

vm（虚拟机保护）是一种基于虚拟机的代码保护技术。他将基于x86汇编系统中的可执行代码转换为字节码指令系统的代码。来达到不被轻易篡改和逆向的目的。

简单来说就是出题人通过实现一个小型的虚拟机，把程序的代码转换为程序设计者自定义的操作码（opcode）然后在程序执行时通过解释操作码，执行对应的函数，从而实现程序原有的功能。

下图是一个一般虚拟机结构

对于上图出现的概念的解释:

VMRUN:虚拟机入口函数
dispatcher：调度器，用于解释opcode，并选择对应的Handler函数执行，当Handler执行完后会跳回这里，形成一个循环。
opcode：程序可执行代码转换成的操作码
Handler：各种功能对象模块

当然这里的虚拟机并不是指VMWare\VirtualBox之类的虚拟机，它更像是一个用于解释系统函数的一个小型模拟器。

3. Pwn中

vm初探 – Wz0beu の Blog

分析思路

  • VM在程序中模拟了stack、text等常见的结构，通过执行指令来模拟功能，我们首先要逆向出各部分的作用。

  • 然后就是找到漏洞，在VM中大多数的漏洞都是由越界读和越界写造成的，所以我们需要密切关注程序中的索引。

  • 最后当我们找到漏洞之后就可以根据漏洞编写我们的exp了，比如我们可以通过覆盖got表来让程序执行后门函数。

题目分析

选择了2019年全国大学生信息安全竞赛的一道题目

由上面这张图我们可以看到，程序的保护主要有四种。

第一种是RELRO，下面对应的是它的开启情况，got表可写，这个got表大家可以理解为程序存储函数地址的一张表。

第二种是Canary，是为了防止栈溢出。

第三种是NX，用于阻断shellcode执行。

最后是PIE，可以随机化内存地址布局，确保每次运行的程序的地址都不相同，不过这里并没有开启这个保护。

具体分析

首先就是main函数：

先对程序进行初始化，然后调用三次creat函数分别开辟空间用于模拟data、text、stack，然后引导用户输入程序名称、指令、以及数据，最后通过调用execute函数执行用户输入的指令。

然后就是在main函数中调用的creat函数：

首先创建了一个0x10大小的堆块用于管理后面的数据，然后根据函数参数的值来创建对应大小的堆块，并将这个堆块的信息赋值给第一个堆块，0x10个字节分别是堆块地址、堆块size/8、以及length。

在main函数中还调用了两个关键函数，其中一个是change函数

根据用户输入的汇编转化为对应的十六进制，然后将结果传到后面的execute函数，

execute函数的流程是根据每个十六进制字节进入不同的分支，进而对栈和data执行对应的操作。

接着就是漏洞分析环节，漏洞主要产生在两个用于模拟指令的函数，第一个就是load

实现的功能是根据栈顶的值计算对应的偏移，然后将偏移对应的值push到栈里

第二个就是save

依旧是根据栈顶的值计算偏移，然后将紧邻栈顶的那个值赋值给偏移对应的地址。由于这两个函数在计算偏移时都没有检查索引的范围，导致了越界读和越界写。

最后是漏洞利用：

1：首先两次push+一次save是为了修改stack到一个可控的地址，然后依靠这个地址计算puts_got所在的偏移，计算结果为-21，将这个值push到栈里，接着load的时候就回将puts_got的值push到栈里。

2：这个时候我们有了libc中的地址，接着计算在libc中system函数与puts函数的偏移，将这个偏移push到栈里，使用add操作，计算出system函数的真实地址。

3：然后将puts_got的偏移再次push到栈里，最后再save就可以将puts_got的值更改为system函数的地址。

4：最后程序会调用puts(s)，这个时候实际调用的就是system(s)，我们只需要在刚开始输入s的时候，输入/bin/sh\x00就可以拿到shell。

成功利用！

4. Patch&实际应用

How to attack

puts(“/bin/sh”) == system(“/bin/sh”)

How to patch

PATCH任意读写

对于vm的攻击大多数需要任意读写，对任意读写的防范也就成了防护VM的重点

所以我们可以增加输入的条件检查：if ( len > (*(int *)(a1 + 12) + v2) > 0)

PATCH漏洞利用

另一个漏洞是因为got表的未保护，导致命令任意写，所以我们开启地址随机化（PIE）保护，禁止写GOT表。
