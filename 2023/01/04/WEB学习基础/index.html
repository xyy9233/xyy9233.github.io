web页面如何展现：

当你在浏览器中输入一个网址（URL）时，浏览器会向对应的 Web 服务器发送一个 HTTP 请求。Web 服务器收到请求后，会根据请求的 URL 返回对应的 Web 页面。这个 Web 页面通常是用 HTML 代码编写的，HTML 是一种用来描述网页结构的标记语言。

浏览器收到服务器返回的 HTML 代码后，会根据 HTML 代码中的标记来渲染出网页的布局和内容。HTML 代码中包含的标记可以描述网页中的文本、图片、表单、视频等元素，浏览器会根据这些标记在网页中显示出对应的内容。

此外，HTML 代码中还可以包含 CSS（层叠样式表）和 JavaScript 代码。CSS 可以用来控制网页的样式和布局，JavaScript 可以用来控制网页的交互效果。

http中get和post区别：

GET 方法用于获取服务器上的资源。它的请求参数会放在 URL 的查询字符串中，所以它只能用来传输少量的数据。

POST 方法用于向服务器发送数据，常用于提交表单或上传文件。POST 方法的请求参数会放在请求体中，所以它可以传输大量的数据

用 wireshark 对 http流量进行分析：

  1. 启动 Wireshark，并选择要分析的网络接口。
  2. 在过滤器栏中输入 “http”，只显示 HTTP 流量。
  3. 在流量列表中选择一条 HTTP 流量，在右侧的窗口中可以看到这条流量的详细信息。
  4. 在分析视图中，可以查看流量的各个部分，例如 HTTP 头、请求体、响应体等。
  5. 在 “Follow Stream” 视图中，可以看到整个 HTTP 流量的内容，并对其进行分析。

如果你想用浏览器控制台分析 HTTP 流量，可以按照以下步骤操作：

  1. 打开浏览器的控制台（一般是通过 F12 键打开）。

  2. 在控制台的 “Network” 选项卡中，可以看到浏览器和服务器之间的所有 HTTP 请求和响应。

  3. 在请求列表中选择一个请求，在右侧的窗口中可以看到这个请求的详细信息，包括 HTTP 头、请求体、响应体等。

  4. 在 “Headers”、”Preview”、”Response”

  5. 项卡中可以看到更多的详细信息，例如 HTTP 头的内容、响应的内容预览、响应的原始内容等。

    在浏览器控制台中，还有很多其他的功能可以帮助你分析 HTTP 流量，例如可以查看网络慢的请求、查看请求的返回状态码、在请求和响应之间添加断点等。

cookie 和 session是怎样进行身份认证的;

https://blog.csdn.net/IT_Holmes/article/details/123360139

Cookie 是浏览器和服务器之间用来存储少量数据的工具。当用户访问一个网站时，服务器可以向浏览器发送一个带有数据的 Cookie，浏览器会将这个 Cookie 存储在本地。当用户再次访问这个网站时，浏览器会将这个 Cookie 发送给服务器，服务器可以根据这个 Cookie 中的数据来判断用户的身份。

Session 也是用来进行身份认证的，但它的实现方式有些不同。当用户访问一个网站时，服务器会为这个用户生成一个唯一的 Session ID，并将这个 Session ID 发送给浏览器。浏览器会将这个 Session ID 存储在本地，并在每次向服务器发送请求时将这个 Session ID 作为参数发送给服务器。服务器会根据这个 Session ID 来判断用户的身份，并为用户提供对应的服务。

Cookie 和 session 都可以用来进行身份认证，但它们有一些区别：

  • 存储位置不同。Cookie 是存储在浏览器端的，而 session 则是存储在服务器端的。
  • 数据大小不同。Cookie 的数据大小通常只有 4KB 左右，而 session 的数据大小没有限制。
  • 有效期不同。Cookie 可以设置过期时间，在超过这个时间后会自动失效；而 session 一般只在浏览器关闭后才会失效。

https 是什么,它与 http 有什么不同;

HTTPS和HTTP的区别

通过上线详细对HTTPS的分析，我们可得：

（1）HTTPS是密文传输，HTTP是明文传输；

（2）默认连接的端口号是不同的，HTTPS是443端口，而HTTP是80端口；

（3）HTTPS请求的过程需要CA证书要验证身份以保证客户端请求到服务器端之后，传回的响应是来自于服务器端，而HTTP则不需要CA证书；

（4）HTTPS=HTTP+加密+认证+完整性保护。

102：继续处理 由WebDAV（RFC 2518）扩展的状态码，代表处理将被继续执行。

200：请求成功 处理方式：获得响应的内容，进行处理

201：请求完成，结果是创建了新资源。新创建资源的URI可在响应的实体中得到 处理方式：爬虫中不会遇到

202：请求被接受，但处理尚未完成 处理方式：阻塞等待

204：服务器端已经实现了请求，但是没有返回新的信 息。如果客户是用户代理，则无须为此更新自身的文档视图。 处理方式：丢弃

300：该状态码不被HTTP/1.0的应用程序直接使用， 只是作为3XX类型回应的默认解释。存在多个可用的被请求资源。 处理方式：若程序中能够处理，则进行进一步处理，如果程序中不能处理，则丢弃

301：请求到的资源都会分配一个永久的URL，这样就可以在将来通过该URL来访问此资源 处理方式：重定向到分配的URL

302：请求到的资源在一个不同的URL处临时保存 处理方式：重定向到临时的URL

304：请求的资源未更新 处理方式：丢弃，使用本地缓存文件

400：非法请求 处理方式：丢弃

401：未授权 处理方式：丢弃

403：禁止 处理方式：丢弃

404：没有找到 处理方式：丢弃

500：服务器内部错误 服务器遇到了一个未曾预料的状况，导致了它无法完成对请求的处理。一般来说，这个问题都会在服务器端的源代码出现错误时出现。

501：服务器无法识别 服务器不支持当前请求所需要的某个功能。当服务器无法识别请求的方法，并且无法支持其对任何资源的请求。

502：错误网关 作为网关或者代理工作的服务器尝试执行请求时，从上游服务器接收到无效的响应。

503：服务出错 由于临时的服务器维护或者过载，服务器当前无法处理请求。这个状况是临时的，并且将在一段时间以后恢复。

网站数据存在哪里,网站是如何传输并存储数据的;

网站的数据通常会存储在服务器端的数据库中。服务器端的数据库是一种特殊的软件，用来存储、组织和管理数据。服务器端的数据库可以帮助网站快速地读写数据，从而提高网站的性能。

网站的数据是通过网络传输和存储的。当用户访问一个网站时，浏览器会向服务器发送请求。服务器收到请求后，会从数据库中读取数据，并将数据返回给浏览器。浏览器收到数据后，会将数据呈现给用户。

当用户在网站上提交数据时，浏览器会将数据发送给服务器。服务器收到数据后，会将数据存储到数据库中。

总之，网站的数据通常会存储在服务器端的数据库中，并通过网络进行传输和存储。网站的数据可以用来记录用户的信息、订单信息、商品信息等，是网站运作的基础。

在网站开发过程中，数据库设计是一个重要的环节。好的数据库设计可以帮助网站快速地读写数据，从而提高网站的性能。数据库设计时，需要考虑的因素包括数据类型、数据结构、索引、数据完整性等。

总之，网站的数据通常会存储在服务器端的数据库中，并通过网络进行传输和存储。网站的数据是网站运作的基础，好的数据库设计可以帮助网站提高性能。

网页的动作控制依靠的是什么,形状和布局依靠的又是什么;

网页的动作控制通常依靠脚本语言，常见的脚本语言包括 JavaScript、VBScript 等。脚本语言是一种编程语言，用来为网页添加交互功能和动态效果。脚本语言通常会运行在浏览器端，可以帮助网页实现动态操作、弹出提示框、执行计算等。

网页的形状和布局通常依靠 HTML 和 CSS。HTML 是网页的结构语言，用来定义网页的结构和内容。CSS 是网页的样式语言，用来定义网页的样式和布局。HTML 和 CSS 可以配合使用，帮助网页设计出美观的形状和布局。

总之，网页的动作控制依靠脚本语言，而形状和布局依靠 HTML 和 CSS。脚本语言帮助网页添加交互功能和动态效果

服务器是什么,如何操作

服务器是一台电脑，用来提供网络服务的计算机。服务器的功能通常包括处理客户端的请求、存储数据、运行程序等。服务器可以运行各种服务软件，比如 Web 服务器、数据库服务器、邮件服务器等，从而为客户端提供服务。

服务器的操作方式可能因服务器的类型、软件环境、网络架构等因素而有所不同。一般来说，服务器可以通过远程桌面连接进行操作，也可以通过 SSH、Telnet 等协议进行远程控制。服务器还可以通过本地控制台进行操作，但这种方式一般只适用于本地管理员。
